## ğŸ¨ Styling Architecture

### Tailwind CSS Approach

This project uses **utility-first CSS** with Tailwind.

**Traditional CSS:**
```css
.button {
  background-color: blue;
  color: white;
  padding: 1rem;
  border-radius: 0.5rem;
}
```

**Tailwind CSS:**
```jsx
<button className="bg-blue-500 text-white p-4 rounded-lg">
  Click Me
</button>
```

### Custom Theme Configuration

Location: `tailwind.config.js`

```javascript
theme: {
  extend: {
    colors: {
      primary: '#00E5FF',      // Electric Cyan
      secondary: '#1E3A5F',    // Navy Blue
      accent: '#C8956D',       // Copper Bronze
      // ... more colors
    }
  }
}
```

### Responsive Design

Tailwind uses breakpoint prefixes:

```jsx
<div className="
  w-full          // Full width on mobile
  md:w-1/2        // Half width on tablets (768px+)
  lg:w-1/3        // Third width on desktop (1024px+)
">
  Content
</div>
```

**Breakpoints:**
- `sm`: 640px
- `md`: 768px
- `lg`: 1024px
- `xl`: 1280px
- `2xl`: 1536px

## âš¡ Key Concepts

### 1. React Hooks

**useState** - Manage component state
```jsx
const [count, setCount] = useState(0);
// count = current value
// setCount = function to update value
```

**useEffect** - Run code when component loads or updates
```jsx
useEffect(() => {
  // This runs when component mounts
  console.log("Component loaded!");
}, []); // Empty array = run once
```

### 2. Props

Pass data from parent to child component:

```jsx
// Parent
<ChannelCard 
  title="About" 
  description="Learn more about me"
  icon={<User />}
/>

// Child (ChannelCard.jsx)
function ChannelCard({ title, description, icon }) {
  return (
    <div>
      {icon}
      <h3>{title}</h3>
      <p>{description}</p>
    </div>
  );
}
```

### 3. Conditional Rendering

Show/hide elements based on conditions:

```jsx
{project.image && (
  <img src={project.image} alt={project.title} />
)}
// Only shows image if project.image exists
```

### 4. Array Mapping

Render lists of items:

```jsx
{projects.map(project => (
  <ProjectCard 
    key={project.id}  // Unique key required
    project={project} 
  />
))}
```

**What happens:**

## ğŸ“ Best Practices

### Component Design

âœ… **Do:**
- Keep components small and focused
- Use descriptive names
- Add comments for complex logic
- Extract reusable parts

âŒ **Don't:**
- Put everything in one file
- Use vague names like `Component1`
- Skip comments on complex code
- Repeat code (DRY principle)

### State Management

âœ… **Do:**
- Keep state as local as possible
- Use meaningful state variable names
- Update state immutably

âŒ **Don't:**
- Put all state in App.jsx
- Mutate state directly
- Over-use state (some things can be calculated)

### Performance

âœ… **Do:**
- Optimise images (use WebP format)
- Lazy load components if needed
- Use proper React keys in lists

âŒ **Don't:**
- Load huge images
- Create functions inside render
- Forget keys in mapped arrays

## ğŸ› Debugging Tips

### Common Issues

**"Module not found"**
- Run `npm install` to install dependencies

**"Port 5173 already in use"**
- Stop other dev servers or use different port

**Changes not appearing**
- Hard refresh browser (Ctrl+Shift+R)
- Check console for errors

**Styling not working**
- Check Tailwind class names (typos?)
- Verify `tailwind.config.js` is correct
